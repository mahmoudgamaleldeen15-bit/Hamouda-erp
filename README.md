# 🚀 دليل استخدام Hamouda ERP v3.0

## 📋 الميزات الجديدة في هذا الإصدار

### ☁️ 1. الرفع اللحظي للسحابة (Firebase Cloud Sync)

النظام دلوقتي بيرفع كل بياناتك على السحابة تلقائياً، وتقدر تفتح النظام من أي جهاز وتلاقي بياناتك.

#### كيف يعمل:

**✅ رفع تلقائي:**
- بعد كل فاتورة/دفعة/مرتجع → **يترفع تلقائياً**
- كل 5 دقايق → **مزامنة كاملة**
- عند فتح النظام → **تحميل آخر بيانات**

**✅ رفع يدوي:**
- زر ☁️ في الهيدر (فوق) → اضغطه في أي وقت للمزامنة الفورية
- تاب "☁️ لم يتم الرفع" في القائمة الجانبية

**✅ عند انقطاع النت:**
- البيانات بتتحفظ محلياً
- Badge أحمر يظهر بعدد العمليات المعلقة
- لما النت يرجع → **الرفع تلقائي**

### 🔢 2. نظام أرقام الفواتير الجديد

كل مستخدم عنده كود (2-3 حروف) بيدخل في رقم الفاتورة:

```
PUR-2026-H-000001   → فاتورة شراء رقم 1 عملها الحاج
SAL-2026-M-000005   → فاتورة بيع رقم 5 عملها المحاسب
RTS-2026-H-000002   → مرتجع بيع رقم 2 عمله الحاج
```

**المميزات:**
- كل مستخدم عنده أرقامه (بدون تعارض بين الأجهزة)
- من الرقم تعرف مين اللي عمل الفاتورة
- يعمل offline بدون مشاكل

## ⚙️ الإعدادات

### تاب "☁️ الرفع اللحظي للسحابة" في الإعدادات:

1. **⚙️ الرفع التلقائي:** اختار كم دقيقة (5 / 10 / 15 / 30 / 60 / إيقاف)
2. **🚀 تحديث تلقائي عند فتح النظام:** ON/OFF
3. **📤 رفع تلقائي بعد كل فاتورة:** ON/OFF
4. **📊 إحصائيات:** كم عملية اترفعت، اتحملت، أخطاء
5. **☁️ رفع وتحميل الآن:** زر للتحديث اليدوي

## 👥 إضافة مستخدمين جدد

عند إضافة مستخدم جديد، **لازم تحدد كود المستخدم**:
- حرف واحد أو حرفين إنجليزي
- Uppercase (H, M, W, S, ...)
- ما يتكررش مع مستخدم تاني

**أمثلة للأكواد:**
- H → الحاج (الأدمن)
- M → المحاسب
- W → مسؤول المخازن
- S → المندوب

## 🚨 حل المشاكل

### "لا يوجد اتصال" رغم أن النت شغال:
- افتح تاب "☁️ لم يتم الرفع"
- اضغط "🩺 تشخيص"
- شوف نتيجة الفحوصات

### الفواتير المعلقة كتيرة:
- افتح تاب "☁️ لم يتم الرفع" 
- اضغط "☁️ ارفع الكل"

### فاتورة معينة مش بترفع:
- افتح تاب "☁️ لم يتم الرفع"
- شوف الفاتورة → اضغط 🔄 (Retry)
- لو استمرت المشكلة → 🗑️ (إزالة من القائمة)

## 📱 استخدام النظام من أكتر من جهاز

**السيناريو المثالي:**
- الحاج على جهازه (كود H)
- المحاسب على جهازه (كود M)
- كل واحد يشتغل عادي
- كل 5 دقايق (أو عند الضغط ☁️) البيانات تتزامن

**ملاحظة مهمة:**
لو الاتنين عدلوا نفس العميل في نفس اللحظة، آخر واحد بيرفع بيفوز (Last Write Wins). ده مقبول لأن الأرقام بـ prefix فمفيش تعارض في الفواتير.

## 🛡️ الأمان

- الباسوردات مشفرة (SHA-256 + Salt) في Firebase
- كل مستخدم بيسجل دخول باليوزر والباسورد المحلي
- Firebase Realtime Database محمي بـ Rules
